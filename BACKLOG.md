# Workout Tracker — Бэклог и План развития (Roadmap)

Этот файл содержит список задач и идей для будущего развития приложения. Если вы открыли новый чат с ИИ-ассистентом, просто попросите его прочитать этот файл (`BACKLOG.md`), чтобы он понял текущий контекст и какие задачи стоят на очереди.

---

## 🌟 Фичи для Пользователей (MVP WOW-эффект)

1. **Onboarding (Приветствие)**
   - **Проблема:** Сейчас новый пользователь видит пустой экран с кнопкой "Создать программу".
   - **Решение:** Сделать красивый пустой экран (Empty State) с приветствием, описанием фич и большой пульсирующей кнопкой "Создать первую тренировку!".

2. **Социальный шеринг (Хвастаемся результатами)**
   - **Проблема:** Тренировки — это социально. Друзья захотят скинуть свои результаты в Telegram.
   - **Решение:** Добавить кнопку "Поделиться" на экране завершения тренировки. При нажатии генерируется красивый текст для мессенджеров (например: `🔥 Завершил тренировку "День ног"! Подходов: 24. Тоннаж: 5600 кг. Сгенерировано в Workout Tracker.`).

3. **Лента истории (Workout Log)**
   - **Проблема:** Нет возможности посмотреть конкретные данные прошлых тренировок.
   - **Решение:** Добавить вкладку "История" в профиль, где в виде ленты будут карточки прошедших тренировок с датами и списком выполненных упражнений.

4. **PWA Лоск (Иконки и сплеш-скрины)**
   - **Проблема:** Приложение нужно красиво устанавливать на экран телефона.
   - **Решение:** Настроить правильный `manifest.json`, добавить красивую иконку (192x192 и 512x512), задать цвет `theme_color` и сплеш-скрин загрузки.

5. **Геймификация (Ачивки)**
   - **Описание:** Выдавать бейджи в профиле: "Первая кровь" (1 тренировка), "Стабильность" (3 тренировки подряд), "Грузчик" (Тоннаж > 10,000 кг).

---

## ⚙️ Технический Долг (Архитектура)

1. **Глобальный рефакторинг состояния (AppState)**
   - **Проблема:** Сейчас переменные (`userPrograms`, `activeProgId`, стейт таймеров и интерфейса) разбросаны глобально по `app.js`.
   - **Решение:** Аккуратно и безопасно обернуть все глобальные переменные в единый объект `AppState` и создать безопасный класс/обертку `Storage` для работы с `localStorage` (с обработкой `try/catch`), чтобы избежать падений приложения при повреждении данных.
   - *Примечание:* Ранее была попытка внедрить это в версии `0.4.3`, но из-за бага в регулярном выражении скрипт затер часть логики, и пришлось делать `revert` до версии `0.4.4`. Следующая попытка должна быть сделана без регулярных выражений, с аккуратной ручной заменой.
